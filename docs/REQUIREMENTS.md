# PTtool 需求文档

## 项目使命

PT 用户在不中断做种的前提下，管理媒体文件目录结构，最大化节约存储空间。

## 项目目标

1. **节约空间**：通过硬链接使多份文件引用同一份数据
2. **保持做种**：不移动/删除源文件，不影响 PT 站点做种
3. **支持刮削**：小文件直接复制，允许 Emby/Jellyfin/tmm 自由修改
4. **幂等安全**：重复运行不产生重复操作，不损坏已有数据
5. **全平台兼容**：Linux、BSD、macOS、Synology、QNAP 等

## 用例

### UC-1：PT 下载完成后自动整理

```
触发：qBittorrent 完成下载
前置：PTtool 已部署、qBittorrent 已配置钩子
流程：
  1. autolink.sh 被 "%N" "%D" "%L" 参数调用
  2. 判断多文件还是单文件种子
  3. 删除下载期间可能被误创建的 islinked.lk
  4. 调用 dirlink.sh 执行硬链接
后置：媒体文件在 DST 目录可用，源文件继续做种
```

### UC-2：批量历史数据整理

```
触发：用户手动执行
前置：已有大量下载完成的 PT 数据
流程：
  1. 用户编辑 link.sh 配置 SRC/DST
  2. 执行 link.sh
  3. 对每个子目录运行 dirlink.sh
后置：所有历史数据被硬链接到 DST
```

### UC-3：单次目录硬链接

```
触发：用户需要一次性链接一个目录
流程：mklink.sh sourcedir dstdir
后置：文件按大小策略被复制或硬链接
```

### UC-4：剧集重命名

```
触发：用户需要调整 SXXEXX 编号偏移
流程：rename_episodes.py <dir> <season_off> <episode_off>
后置：目录中的文件按偏移量重命名
```

### UC-5：跨文件系统检测

```
触发：用户执行 mklink.sh 或 dirlink.sh
前置：SRC 和 DST 在不同分区
流程：
  1. 脚本检查 SRC 和 DST 是否在同一文件系统
  2. 如果在不同分区，打印明确错误信息并退出
后置：操作中止，用户收到"硬链接不能跨文件系统"提示
```

### UC-6：符号链接处理

```
触发：SRC 目录中包含符号链接
前置：目标目录中存在指向外部文件的符号链接
流程：
  1. find 默认跟随符号链接，可能导致链接到外部路径
  2. 脚本使用 -type f 确保只处理普通文件
后置：符号链接目录被跳过，仅处理真实文件
```

### UC-7：增量补充链接

```
触发：用户向已链接的目录中添加了新文件
前置：目录已有 islinked.lk 标记
流程：
  1. dirlink.sh 检测到 islinked.lk 会跳过整个目录
  2. 用户需手动删除 islinked.lk 后重新运行
后置：新文件被链接，旧文件维持现有状态
```

## 非功能需求

| 需求 | 指标 | 优先级 |
|------|------|--------|
| POSIX 兼容 | 在 dash/busybox ash/ksh 下无语法错误 | P0 |
| 平台兼容 | Linux/BSD/macOS/Synology/QNAP | P0 |
| 幂等性 | 重复运行结果一致 | P0 |
| 零外部依赖 | 仅使用 POSIX 标准工具 + curl (autoThanks.sh) | P0 |
| 无数据破坏 | 不使用 rm（除 islinked.lk） | P1 |
| 可重入 | 任何步骤中断后重跑不损坏数据 | P1 |
| 错误信息明确 | 跨文件系统、参数缺失等可操作错误提示 | P0 |
| 退出码规范 | 成功 0，参数错误 1，运行时错误 2 | P1 |
| 安全性 | 不执行 rm（除 islinked.lk），不使用 eval | P1 |
| 性能 | 单次运行不超过 5 分钟（100GB 量级） | P2 |
